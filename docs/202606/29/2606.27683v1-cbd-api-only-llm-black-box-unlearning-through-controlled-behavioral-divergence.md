## 摘要
边缘设备越来越多地通过API服务调用大语言模型（LLM）以实现上下文感知的边缘智能，而边缘生成的数据可能被收集用于改进LLM，并可能将敏感、有版权、有害或过时的信息引入模型行为。机器反学习提供了一种无需重新训练LLM即可移除不良数据影响的实用方法。然而，现有方法仍面临两个差距。其一是仅API的黑盒访问，其中目标模型参数和内部logits不可用。其二是当反学习目标数据与保留数据共享高度相似的提示结构或语义模式时，如何保持保留效用。为解决这些挑战，我们提出了受控行为差异（CBD），一种仅API的黑盒反学习框架。CBD使用两个辅助模型在保留输入和反学习目标输入之间创建受控行为差异，将此差异转换为反学习相关性分数，并将反学习相关提示路由远离目标LLM。为了提高在目标数据与保留数据高度相似情况下的区分准确性，CBD通过估计经验Fisher矩阵并求解正则化广义特征值问题，构建基于梯度统计的判别基，引导反学习信号朝向目标特定信息而非共享提示结构。与11种白盒和灰盒反学习基线相比，CBD实现了更好的反学习-效用权衡，且其性能在不同设置下波动很小。在ToFU forget10上，CBD接近重训练的参考模型在遗忘集上的表现，同时将模型效用提升至74.90，比次优基线高出约15%。在WMDP上，它将有害知识准确率降至25.68，接近随机猜测，同时保持MMLU准确率为52.67。代码位于https://github.com/DGL-codes/CBD。

## Abstract
Edge devices increasingly invoke large language models (LLMs) through API services for context aware edge intelligence, while edge generated data may be collected to improve LLMs and may introduce sensitive, copyrighted, harmful, or outdated information into model behavior. Machine unlearning offers a practical way to remove the influence of undesired data without retraining LLMs. However, existing methods still face two gaps. The first is API only black box access, where target model parameters and internal logits are unavailable. The second is how to preserve retained utility when unlearning target data and retained data share highly similar prompt structures or semantic patterns. To address these challenges, we propose Controlled Behavioral Divergence (CBD), an API only black box unlearning framework. CBD uses two auxiliary models to create controlled behavioral divergence between retained inputs and unlearning target inputs, converts this divergence into an unlearning relevance score, and routes unlearning related prompts away from the target LLM. To improve discrimination accuracy under high similarity between target and retained data, CBD constructs a gradient statistics based discriminative basis by estimating empirical Fisher matrices and solving a regularized generalized eigenvalue problem, guiding the unlearning signal toward target specific information rather than shared prompt structures. Compared with eleven white box and gray box unlearning baselines, CBD achieves a better unlearning utility trade off and its performance varies little across settings. On ToFU forget10, CBD approaches the retrained reference on the forget set while raising model utility to 74.90, about 15% above the second best baseline. On WMDP, it lowers hazardous knowledge accuracy to 25.68, near random guessing, while preserving MMLU accuracy of 52.67. Code is at https://github.com/DGL-codes/CBD.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **研究动机**：边缘设备通过API调用大语言模型（LLM）时，用户查询等边缘数据可能被收集以改进基础模型，这些数据可能包含敏感、有版权、有害或过时的信息。为移除这些不良数据的影响，机器反学习成为关键方法，但现有方法面临两个核心挑战：
  1. **API-only黑盒访问**：在边缘服务中，目标LLM仅通过API返回最终响应，无法获取模型参数、梯度或内部logits，使白盒方法（需更新模型参数）和灰盒方法（需修正logits）无法适用。
  2. **高相似性下的保留效用保持**：遗忘数据和保留数据常共享高度相似的提示模板或语义结构（仅实体或属性不同），导致反学习信号易作用于共享结构而非目标特异性内容，损害保留数据上的模型性能。
- **整体含义**：本文提出CBD（Controlled Behavioral Divergence），首个**仅需API黑盒访问**的反学习框架，旨在不修改目标LLM、不访问其内部状态的前提下，通过路由机制减少遗忘数据的暴露，并解决高相似性下区分困难的问题。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（文字说明）
- **核心思想**：利用两个辅助模型（冻结的参考模型 `M_ref` 和可训练的探测模型 `M_pro`）在保留输入上保持行为一致，在对遗忘输入上产生行为分歧，将分歧转化为遗忘相关性分数，并将高分数查询路由到参考模型（而非目标LLM）响应，从而减少目标LLM对遗忘数据的输出。
- **关键技术细节**：
  - **双辅助模型机制**：参考模型 `M_ref` 保持初始参数不变，探测模型 `M_pro` 通过LoRA适配器训练，优化目标为在遗忘集上增大负对数似然（梯度上升），在保留集上最小化与参考模型的序列级KL散度（保持行为一致）。
  - **行为分歧得分**：利用对称KL散度计算两个辅助模型在输入上的分布差异，聚合得到遗忘相关性得分 `s(x)`，通过验证集搜索阈值 `τ`，路由规则：若 `s(x) > τ` 则用 `M_ref` 回答，否则用目标LLM回答。
  - **判别Fisher基（核心贡献）**：针对高相似性场景，构建基于梯度统计的判别基，引导探测模型更新方向最大化遗忘变化同时最小化保留变化。
    - 估计遗忘侧和保留侧的经验Fisher矩阵：`F_f = (1/N_f) G_f G_f^T`，`F_r = (1/N_r) G_r G_r^T`。
    - 引入阻尼因子 `μ`，定义 `C = F_r + μI`。
    - 求解正则化广义特征值问题：`F_f v = λ C v`，取前k个广义特征向量构成判别基 `Q`。
    - 训练时，将遗忘梯度投影到判别基上：`\bar{g}_f = Q Q^T g_f`，避免抑制保留方向的同时保持遗忘分离。
  - **算法流程（Algorithm 2）**：
    1. 用Algorithm 1提取判别基 `Q`。
    2. 初始化 `M_pro` 参数为 `w_0`。
    3. 迭代训练：计算遗忘梯度 `g_f` 和保留梯度 `g_r`，投影遗忘梯度 `\bar{g}_f = Q Q^T g_f`，组合梯度 `g = \bar{g}_f + β g_r`，更新参数。
    4. 基于验证集得分选择路由阈值 `τ`。
    5. 部署时根据分数路由查询。

## 3. 实验设计：数据集/场景、基准（benchmark）、对比方法
- **数据集**：
  - **ToFU**（虚构作者问答）：包含4000个虚构作者QA对、100个真实作者问题、117个世界事实问题，定义三个遗忘比例设置：forget01（1%）、forget05（5%）、forget10（10%）。
  - **WMDP**（大规模杀伤性武器代理基准）：包含3,668个有害知识选择题（生物、网络安全、化学），保留效用评估用MMLU（14,042题，57科目）。
- **基准模型**：标准参考包括原始目标LLM和重训练LLM（仅ToFU）。
- **对比方法**：
  - **白盒方法**：GA（梯度上升）、DPO（直接偏好优化）、NPO（负偏好优化），每种都带有+GD（梯度下降保留）和+KL（KL散度保留）变体。
  - **灰盒方法**：ULD（基于logit差的反学习）、Offset Unlearning（偏移反学习），它们需要目标模型logits。
  - **CBD本身**：黑盒方法（仅需API响应）。
- **辅助模型**：两个辅助模型均初始化为TinyLlama-1.1B-Chat-v1.0（冻结+LoRA可训练）。

- **核心结论**：CBD是首个仅需API黑盒访问就能有效实现LLM反学习的框架，在遗忘效果与保留效用之间取得更优平衡，且性能对超参数变化鲁棒。

## 8. 不足与局限
- **实验覆盖有限**：
  - 仅在ToFU（虚构作者）和WMDP（有害知识）两个数据集上验证，未涉及版权或隐私等更广泛的反学习场景。

  - WMDP未提供重训练参考，仅以随机猜测水平对比，可能高估了遗忘彻底性。
- **概念层面的局限性**：
  - CBD本质上是“路由”而非“擦除”，遗忘数据仍可能存在于目标模型参数中，只是不暴露给用户；对于要求完全移除数据的场景可能不足。

  - 路由策略基于单查询的独立分数，未考虑对话历史或上下文依赖性。
- **实验细节的潜在风险**：
  - 路由阈值在验证集上选择，若验证集分布与真实部署分布偏移，可能影响路由性能。
  - 步稳定性实验仅考察了80-200步，未展示更长训练下的行为。
