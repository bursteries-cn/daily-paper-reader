## 摘要
知识编辑系统必须更新选定的事实，同时保持邻近但无关的行为不变。本文在记忆辅助设置下研究该问题，其中在推理时检索编辑记忆，并通过参数高效适配器修正模型的物体偏好。我们认为核心设计问题不仅在于如何写入编辑，还在于何时抑制编辑。我们引入了\method{}，一种路径专用双适配器编辑器。相关性路由器首先决定是否应对提示接收编辑记忆。路由后的提示使用一个编辑适配器，该适配器训练为偏好新物体而非原始物体；未路由的非直接提示使用一个独立的局部性适配器，训练为保持或恢复原始物体偏好。我们在相同的记忆协议和两种7B/8B基础模型下，在三个1000样本协议\cf{}、\zsre{}和\mquake{}上评估\method{}。在Llama-3.1-8B-Instruct上，\method{}在所有三个基准上取得了最佳的整体概率偏好准确率：\cf{}为0.8180，\zsre{}为0.8946，\mquake{}为0.9922。同样的趋势在Qwen3-8B上保持。路由器消融实验表明，相关记忆边界在不同数据集上有所不同：在\cf{}上词汇神经路由器最安全，而在\zsre{}和\mquake{}上BGE嵌入路由更好。组件和模块消融实验表明，增益主要来自于将编辑注入与脱离路径抑制分离，而非简单地增加LoRA容量。

## Abstract
Knowledge editing systems must update selected facts while preserving nearby but irrelevant behavior. This paper studies this problem in a memory-assisted setting where an edit memory is retrieved at inference time and a parameter-efficient adapter corrects the model's object preference. We argue that the central design question is not only how to write an edit, but also when to suppress it. We introduce \method{}, a route-specialized dual-adapter editor. A relevance router first decides whether a prompt should receive an edit memory. Routed prompts use an edit adapter trained to prefer the new object over the original object; unrouted non-direct prompts use a separate locality adapter trained to preserve or restore the original-object preference. We evaluate \method{} on three 1,000-case protocols, \cf{}, \zsre{}, and \mquake{}, under the same memory protocol and two 7B/8B base models. On Llama-3.1-8B-Instruct, \method{} obtains the best overall probability-preference accuracy on all three benchmarks: 0.8180 on \cf{}, 0.8946 on \zsre{}, and 0.9922 on \mquake{}. The same trend holds on Qwen3-8B. Router ablations show that the relevant memory boundary differs across datasets: a lexical neural router is safest on \cf{}, while BGE embedding routing is better on \zsre{} and \mquake{}. Component and module ablations show that the gain mainly comes from separating edit injection from off-route suppression rather than from simply increasing LoRA capacity.

- **核心问题**：论文认为，记忆辅助编辑器的核心设计问题不仅是“如何写入编辑”，更是“何时抑制编辑”。即需要解决两个关联决策：何时应该检索编辑记忆？检索后哪个修正模块应该激活？

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：引入**RRDA（Relevance-Routed Dual Adapters）**，一个路径专用双适配器编辑器。一个**相关性路由器**决定提示是否应接收编辑记忆。路由后的提示使用**编辑适配器**（训练为偏好新对象而非原对象）；未路由的非直接提示使用**独立的局部性适配器**（训练为保持或恢复原对象偏好）。该设计分离了“写入”和“抑制”两个操作。
- **关键技术细节**：
  - **问题设置**：每个编辑案例由（主体 s，关系 r，原对象 o0，新对象 o⋆）表示。记忆库为每个编辑案例存储一条文本记录。查询提示 x 分为直接、改写或局部三类。使用长度归一化的对数概率 score(x,y) 计算对象偏好。
  - **相关性路由器**：评估四个路由器家族：
    - Subject：查询是否包含编辑主体字符串；
    - Subject+relation：主体匹配加关系-标记重叠；
    - NN：基于哈希词袋的MLP，训练使用正例（相同案例）和负例（随机、局部、硬主体负例）；
    - Semantic：BGE嵌入相似度，可选加MiniLM交叉编码器重排。
    最终选定路由器基于验证：CounterFact使用NN路由；ZsRE和MQuAKE-CF使用BGE嵌入路由。路由器不再是训练决定哪些案例进入记忆，记忆库在评估前固定。
  - **双适配器设计**：
    - **编辑适配器**：处理直接提示和路由后的非直接提示。训练损失：编辑答案交叉熵 + 成对排序损失（L_edit = λ_ce L_ce + λ_rank L_rank，其中L_rank = softplus(m - score(x,y+) + score(x,y-))）。对于直接/改写，(y+, y-) = (o⋆, o0)；对于局部提示（用于局部适配器），偏好反转。
    - **局部性适配器**：仅训练无记忆局部排序示例：L_loc = softplus(m_loc - score(x, o0) + score(x, o⋆))。
    - **推理门控策略**：固定策略：直接提示使用编辑适配器（门控0.75）；非直接路由提示使用编辑适配器（门控0.50）；非直接未路由提示使用局部适配器（门控0.75）。门控值在验证集选定并固定。
