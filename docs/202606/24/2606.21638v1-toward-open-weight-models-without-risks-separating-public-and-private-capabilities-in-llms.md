## 摘要
开放权重的大语言模型（LLM）促进了科学进步和广泛部署。然而，它们也使得对敏感能力的访问控制变得困难。当前的做法要么在发布前抑制危险能力，要么通过使用专用模型变体、输入/输出监控和API权限的封闭服务来中介访问。前者容易受到越狱攻击，同时为了减轻少数用户带来的风险而牺牲了所有用户的能力；后者则与开放权重发布根本不相容。在本文中，我们提出了分层语言模型（TLM），其中一组发布的权重支持多个能力级别。在其默认的公共配置下，TLM的行为与传统LLM相同。一个紧凑的密钥指定了少量参数子集上的排列，从而在相同权重上诱导出替代计算图，暴露出额外能力。我们开发了一种训练协议，从零开始联合预训练两种配置，然后对密钥配置在私有数据上进行微调，并加入正则化以保持公共模型的行为。我们预训练了1.8亿和6.5亿参数的TLM，并展示了密钥配置可以习得新语言、获得指令遵循能力并记忆私有事实知识，而公共配置则不具备这些能力。此外，我们展示了该方法自然地扩展到多个层次级别。由于授权操作基于模型的权重结构而非输入空间，该机制能够抵抗基于微调的提取和部分密钥泄露。总的来说，TLM朝着调和开放权重发布与选择性能力控制迈出了一步。

## Abstract
Open-weight Large Language Models (LLMs) enable scientific progress and broad deployment. However, they make it difficult to control access to sensitive capabilities. Current practice either suppresses dangerous capabilities before release or mediates access through closed services that use specialized model variants, input/output monitors, and API permissions. The former is susceptible to jailbreaks while sacrificing capability for all users to mitigate the risks posed by a few, and the latter is fundamentally incompatible with open-weight release. In this paper, we propose Tiered Language Models (TLMs), where a single set of released weights supports multiple capability levels. In its default public configuration, a TLM behaves as a conventional LLM. A compact secret key specifies a permutation over a small parameter subset, inducing an alternative computation graph over the same weights that exposes additional capabilities. We develop a training protocol that jointly pretrains both configurations from scratch, then fine-tunes the keyed configuration on private data with regularization to preserve the public model's behavior. We pretrain 180M- and 650M-parameter TLMs and demonstrate that the keyed configuration can acquire a new language, gain instruction-following ability, and memorize private factual knowledge, whereas the public configuration exhibits none of these capabilities. Moreover, we show that our approach extends naturally to multiple hierarchical tiers. Because authorization operates on the model's weight structure rather than in the input space, the mechanism resists fine-tuning-based extraction and partial key compromise. In general, TLMs take a step toward reconciling open-weight release with selective capability control.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将授权从输入空间（密码字符串）转移到模型的参数空间——通过一个紧凑的**排列密钥**重新排列少量权重的位置，从而在不改变权重数值的前提下，诱导出不同的计算图（私密配置），公开配置则保持原样。
- **关键技术细节**：
  1. **定义**：模型具有公共配置 `C_pub = identity` 和一个或多个密钥配置 `C_K`（通过密钥指定排列）。被排列的参数子集 `S` 称为“层级参数”，约占模型总参数的 5%，包括两类：
     - 注意力头分组：不同层的注意力头之间交换 Q/K/V 行和输出投影列。
     - FFN 分组：不同层的 MLP 神经元之间交换上下投影行/列。
  2. **训练协议（两阶段）**：
     - **第一阶段：非对称联合预训练**。在同一公开语料 `D_pub` 上同时训练两种配置，但梯度流不对称：层级参数 `S` 仅从密钥配置接收梯度（`∇θ_S L_pre = E[∇θ_S ℓ( p_{C_K(θ)} , y)]`）；互补参数 `\bar{S}` 接收两者混合梯度（`∇θ_{\bar{S}} L_pre = E[λ1 ∇θ_{\bar{S}} ℓ( p_{C_pub(θ)} , y) + λ2 ∇θ_{\bar{S}} ℓ( p_{C_K(θ)} , y)]`）。这使得公共配置必须学习依赖 `\bar{S}` 来完成任务，从而对未来 `S` 的更新具有鲁棒性。
     - **第二阶段：私有微调与正则化**。冻结 `\bar{S}`，仅更新 `S` 并通过密钥配置计算私有数据 `D_priv` 上的交叉熵损失。同时添加 KL 散度正则化项 `R_KL = E_{x∼D_pub}[KL( p_{C_pub(θ_pre)} ∥ p_{C_pub(θ)})]`，防止公共行为漂移。最终目标：`L_ft(θ_S) = L_priv(θ_S) + β R_KL(θ_S)`。
  3. **多层级扩展**：支持顺序累积的多个密钥（每个密钥作用于不相交的子集 `S_i`）。预训练时轮询不同密钥配置，微调时顺序进行，并添加对先前层级的保真正则化项。

## 6. 论文的主要结论与发现

- **能力分离成功**：在所有三个任务中，密钥配置（`C_K`）成功习得私有能力（西班牙语损失下降到 3.15 附近，AlpacaEval 获胜率超 85%，事实记忆精确匹配达 100%），而公开配置（`C_pub`）完全没有表现出这些能力（损失不变、获胜率约 15%、精确匹配为 0）。
- **额外训练成本低**：密钥更新频率 `f=20` 即可达到近乎全频率的性能，额外预训练 FLOPs 仅约 5%。TLM 的公开配置仅比非层级基线多约 6% 的训练步数。
- **密钥极紧凑**：5% 密钥在信息理论上比匹配性能的 1% LoRA 适配器小 560 倍（180M 模型）到 7000 倍以上（100B 模型）。
- **鲁棒性强**：
  - 即使在公开配置上微调一半的私有数据，也无法提取出另一半隐藏事实。
  - 即使知道 90% 的密钥排列，能力恢复仍接近零（精确匹配）。这意味着密钥更像密码而非软控制。
  - 权重幅度可部分识别层级参数（F1 ~54%），但不足以恢复排列本身。
- **多层级有效**：三层级 TLM 可以顺序微调不同语言（德语、土耳其语、西班牙语），各层级能力保持独立且公开配置不受影响。

## 7. 优点：方法或实验设计上的亮点

- **创新性框架**：首次将模型级权限控制建立在参数排列而非输入提示或外部适配器基础上，兼具开放权重和选择性能力控制。
- **极致的紧凑性**：密钥仅存储排列索引而非学习参数，存储和传输成本极低。
- **训练协议巧妙**：非对称联合预训练迫使公共配置依赖非层级参数，从而对后续层级参数更新免疫。
- **鲁棒性验证全面**：不仅测试了标准使用场景，还覆盖了微调提取、部分密钥、权重指纹三种攻击，结果清晰。
- **多层级扩展自然**：支持有序授权层次，且保真正则化确保了层级间不退化。

## 8. 不足与局限

- **模型规模限制**：仅验证到 650M 参数，未在数十亿或千亿级模型上实验。前向模型可能存在不同的干扰模式或新的泄漏通道。
- **威胁模型有限**：未考虑结构化排列搜索、激活级分析、结合任务知识的白盒攻击。论文未提供密码学安全保障，部分密钥实验不可视为形式保证。
- **可检测指纹**：私有微调后权重幅度存在统计痕迹（MLP 块中层级参数范数偏小），简单阈值检测可部分识别（F1~54%），可能被更强攻击利用。

- **实际部署挑战**：需要安全的密钥管理基础设施，且对实际敏感能力（如生化知识）的评估仅在代理任务上进行，未在真实危险能力上直接测试。
