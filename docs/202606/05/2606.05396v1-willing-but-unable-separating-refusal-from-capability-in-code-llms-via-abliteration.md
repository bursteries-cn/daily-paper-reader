## 摘要
大规模生成带有标签的漏洞代码是基于学习的漏洞检测的一个反复出现的障碍：挖掘的语料库带有大量标签噪声，而现有的基于LLM的增强方法传播了这些不准确性，因为它转换有漏洞的种子而不是从规范中合成漏洞。一条互补的路线是从安全代码开始，要求经过指令调优的LLM注入指定的CWE（这将把标签负担从开放性检测转移到有界的二元确认），但安全对齐的代码LLM系统地拒绝此类提示。本文是对abliteration的初步可行性研究，abliteration是一种低秩权重编辑，通过正交投影去除残差流中的拒绝方向，作为消除这一障碍的工具。我们使用Python和CWE-89（SQL注入）作为案例研究，评估Qwen2.5-Coder-Instruct家族（3B、7B和14B参数）在来自PromSec和SafeCoder的安全样本上的表现，每种条件重复三次。我们发现：（i）对注入提示的拒绝强烈依赖于大小和提示上下文：14B拒绝100%的提示，7B拒绝PromSec的73%但仅拒绝SafeCoder的5%，而3B基本上从未被阻断；（ii）abliteration将拒绝率降低到零或接近零，同时保持语法有效性高于93%，支持这样的观点：在这种设置下，拒绝与测量的代码生成能力可以分离；（iii）abliteration后的注入率仍然受能力限制（14B为88-97%，7B为89-90%，3B为25-48%），将意愿（abliteration解锁的）与能力（随参数扩展的）分离开。漏洞判定由三个工具检测器集成（CodeQL、Semgrep、Bandit）生成，然后由两位作者对检测器阳性输出进行人工裁决。

## Abstract
Producing a labeled vulnerable code at scale is a recurring obstacle for learning-based vulnerability detection: mined corpora carry substantial label noise, and existing LLM-based augmentation propagates these inaccuracies because it transforms vulnerable seeds rather than synthesising vulnerabilities from a specification. A complementary route is to start from safe code and ask an instruction-tuned LLM to inject a specified CWE (which would shift the labeling burden from open-ended detection to bounded binary confirmation) but safety-aligned code LLMs systematically refuse such prompts. This paper is a preliminary feasibility study of abliteration, a low-rank weight edit that orthogonally projects out the refusal direction in the residual stream, as a tool to remove this barrier. We use Python and CWE-89 (SQL injection) as a case study, evaluating the Qwen2.5-Coder-Instruct family at 3B, 7B, and 14B parameters on safe samples drawn from PromSec and SafeCoder, replicated three times per condition. We find that (i) refusal on injection prompts is strongly size- and prompt-context-dependent: the 14B refuses 100% of prompts, the 7B refuses 73% of PromSec but only 5% of SafeCoder, whereas the 3B is essentially never blocked; (ii) abliteration reduces refusal to zero or near-zero across all sizes while leaving syntactic validity above 93%, supporting the view that, in this setting, refusal can be detached from measured code-generation capability; and (iii) the post-abliteration injection rate remains capacity-bound (88-97% on the 14B, 89-90% on the 7B, and 25-48% on the 3B) separating willingness, which abliteration unlocks, from capability, which scales with parameters. Vulnerability verdicts are produced by a three-tool detector ensemble (CodeQL, Semgrep, Bandit) followed by manual adjudication by two authors on detector-positive outputs.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：基于学习的漏洞检测（如训练神经网络识别漏洞）面临高质量标注数据集匮乏的瓶颈。现有主流方法包括：
  - 从CVE提交记录中挖掘的语料库（如Devign），但存在20–71%的标签噪声和高达99%的重复率。
  - 基于LLM的增强方法（如VulGen、VulScribeR）只能转换已有的漏洞种子，无法从规范合成漏洞，因此继承了种子数据的噪声。
- **互补思路**：从经静态分析确认为安全的代码出发，让指令调优的代码LLM按指定CWE（如CWE-89 SQL注入）进行注入，生成有漏洞的变体。这样标签任务从“开放的多分类检测”简化为“有界二元确认”（仅需确认是否为目标CWE），且预期噪声更低。
- **核心障碍**：安全对齐的代码LLM（通过RLHF/DPO训练）会拒绝明确要求引入漏洞的提示（如“用CWE-89重写”），即使它具备完成该转换的能力。拒绝行为掩盖了模型的实际能力，使得常规评估无法区分“不愿做”与“不会做”。
- **整体含义**：本文验证abliteration（低秩权重编辑）能否有效移除这种拒绝行为，从而解锁模型的内在生成能力，为大规模合成高质量漏洞数据集提供可行路径，同时揭示意愿与能力在代码LLM中的可分离性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：基于Arditi等人（2024）的发现——语言模型的拒绝行为由残差流中一个（或少数几个）方向介导。通过在每一层将这些方向正交投影出权重矩阵，可以永久性地移除拒绝行为，同时保留模型的其他能力。
- **关键技术细节**：
  - **激活收集**：在同一组有害/无害提示上，收集模型每一层最终token位置的残差流激活值，分别得到矩阵 \(H\)（有害）和 \(B\)（无害）。
  - **对比分析**：计算差值矩阵 \(D = H - B\)，进行奇异值分解 \(D = USV^\top\)，取前 \(k\) 个右奇异向量作为候选拒绝方向 \(\hat{r}_1, \dots, \hat{r}_k\)。
  - **权重编辑**：对所有写入残差流的权重矩阵 \(W_{\text{out}}\)（嵌入层、注意力输出投影、MLP输出投影）执行编辑：
    \[
    W'_{\text{out}} \leftarrow W_{\text{out}} - \sum_{i=1}^k \hat{r}_i \hat{r}_i^\top W_{\text{out}}
    \]
    从而阻止模型向残差流写入这些方向。
  - **实施细节**：使用Obliteratus工具包，采用高级方法、正交归一化投影、逐层自适应强度缩放和top-k层选择。不同模型大小超参数不同：3B使用 \(k=2\)、2次精调、针对上层10层；7B使用 \(k=8\)、4次精调、9层；14B使用 \(k=8\)、4次精调、16层。
  - **验证阶段**：Obliteratus内置verify：要求有害提示拒绝率≤5%、无害提示困惑度与基座差异≤0.5、输出Python代码语法有效且语义合理。不满足则重跑。
- **最终产物**：经过4-bit量化（Q4_K_M）的GGUF格式abliterated模型，可在无GPU的通用笔记本电脑上推理。

## 3. 实验设计

- **模型**：Qwen2.5-Coder-Instruct系列，3B、7B、14B参数各两个条件：基座（Base）和Abliterated。

  4. 任何检测器标记为阳性的输出，由两位作者独立人工裁决（CWE-89符合性），分歧通过讨论解决。

## 6. 主要结论与发现

1. **RQ1：拒绝行为依赖模型大小和提示上下文**：
   - 14B基座拒绝所有CWE-89提示（100%），无论数据集。
   - 7B基座在PromSec上拒绝73%，在SafeCoder上仅拒绝5%（因SafeCoder片段短、风险感知低）。
   - 3B基座几乎不拒绝（PromSec 0%, SafeCoder 6.7%）。
2. **RQ2：Abliteration有效移除拒绝而不损害能力**：
   - 所有abliterated模型的拒绝率降至0或接近0（仅3B在PromSec上残留1.7%）。
   - 语法有效性保持在93%以上（3B、7B、14B），与基座模型非拒绝输出的水平一致。
   - KL散度测量（1.51/1.57/1.39）表明对输出分布的扰动在可接受范围内。
   - 手动检查发现，模型仅修改了SQL构造部分，保留了路由、哈希函数等。
3. **RQ3：解离意愿与能力——注入率受模型容量限制**：
   - 移除拒绝后，注入率随模型大小显著上升：3B仅25-48%，7B达89-90%，14B达88-97%。
   - 7B和14B的注入率在PromSec上接近（90% vs 88%），但14B在SafeCoder上略高（96.7% vs 89.3%）。
   - 3B的注入率与基座无显著差异（25% vs 31.7%），表明其能力已达到上限，abliteration不能创造本不存在的知识。
   - **关键洞察**：abliteration解除了“不愿”的抑制，但“不能”的问题需通过更大的模型或更优的训练数据解决。
4. **检测器一致性**：三个检测器（CodeQL、Semgrep、Bandit）的共识率低（仅71/630个输出为全阳性），验证了人工裁决的必要性。
