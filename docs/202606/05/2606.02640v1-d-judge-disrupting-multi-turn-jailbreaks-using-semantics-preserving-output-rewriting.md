## 摘要
多轮越狱攻击对大型语言模型（LLM）的安全性构成日益严重的威胁，因为它们利用来自辅助评判模型的反馈，迭代优化提示以实现有害目标。现有防御措施主要在单个回合或最终响应中检测或阻止不安全内容，但保留了由评判模型驱动的优化循环，使攻击者能够从中间交互中提取信息反馈。我们提出了D-Judge，一种语义保持的输出重写防御方法，该方法直接干预该循环，在攻击者的评判模型评估受害LLM的响应之前对其进行重写。通过在不改变原始响应含义的情况下使评判者的反馈信号失准，D-Judge扰乱了攻击者的提示优化过程，导致后续查询针对扭曲的攻击进展信号进行优化。为了提高D-Judge生成此类重写的能力，我们构建了一个语义等价的响应对数据集，这些响应对会诱导评判模型分配不同的有害性分数，并用于监督微调和直接偏好优化。在HarmBench上的实验表明，D-Judge在保持良性基准性能的同时，降低了最先进的多轮越狱攻击的成功率。

## Abstract
Multi-turn jailbreak attacks pose a growing threat to large language model (LLM) safety because they exploit feedback from auxiliary judge models to iteratively refine prompts toward harmful goals. Existing defenses largely detect or block unsafe content at individual turns or at the final response, leaving the judge-driven refinement loop intact and allowing attackers to extract informative feedback from intermediate interactions. We introduce D-Judge, a semantics-preserving output rewriting defense that intervenes directly in this loop by rewriting the victim LLM's responses before they are evaluated by the attacker's judge. By misaligning the judge's feedback signal without changing the meaning of the original response, D-Judge derails the attacker's prompt-refinement process, causing subsequent queries to be optimized against a distorted signal of attack progress. To improve D-Judge's ability to produce such rewrites, we construct a dataset of semantically equivalent response pairs that induce different judge-assigned harmfulness scores, and use it for supervised fine-tuning followed by direct preference optimization. Experiments on HarmBench show that D-Judge reduces the success rate of state-of-the-art multi-turn jailbreaks while preserving performance on benign benchmarks.

- **核心问题**：多轮越狱攻击（Multi-turn Jailbreak）利用外部评判模型（Judge LLM）的反馈信号，在多个对话轮次中迭代优化恶意提示，逐步引导受害 LLM 生成有害内容。现有防御方法（如单轮检测、多轮上下文风险跟踪、输入/输出预处理等）仅在单个轮次或最终响应处阻止不安全内容，但未能破坏攻击者依赖的“评判反馈循环”，攻击者仍能从中间轮次中获取信息性反馈，持续优化后续查询。
- **整体含义**：该工作揭示了评判驱动优化循环是多轮越狱攻击的关键机制，并提出通过**在反馈信号传输环节进行干预**的防御思路，而非简单地检测或阻断输出。这种范式转变——从“阻止有害内容”转向“破坏攻击者的优化信号”——为 LLM 安全提供了新的视角。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **核心思想**：在受害 LLM 输出被攻击者评判模型评估之前，对输出进行**语义保持的重写**（Semantics-Preserving Rewriting）。重写后的响应保持原意但会**诱导裁判模型给出失真的反馈分数**（如抬高或降低有害性评分）。由于攻击者后续轮次的查询优化依赖于该反馈信号，失真信号将导致优化过程偏离真实攻击进展，从而破坏攻击。
- **关键洞察**：攻击者通常使用条件 \(s_i > \max(s_1,\dots,s_{i-1})\) 来判断是否进入下一轮（渐进式升级）。D-Judge 通过扭曲 \(s_i\) 打破该条件，甚至利用早期停止机制使攻击提前终止。

### 2.2 关键技术细节
1. **系统架构**：D-Judge 位于受害者 LLM 与用户（攻击者）API 之间，包含一个重写器（Rewriter）和一个语义门（Semantic Gate）。
   - 重写器接收原始响应，输出语义等价的改写版本。
   - 语义门使用自然语言推理（NLI）模型进行双向蕴含检查，拒绝破坏语义的改写，必要时重新生成（最多 5 次）。
2. **训练数据构造**：
   - 从 PKU-SafeRLHF 中采样中等有害程度的响应。
   - 使用候选生成 LLM 生成两个方向的重写：`INCREASE`（提高有害性）和 `DECREASE`（降低有害性）。
   - 筛选条件：（a）使用 Llama Guard 计算不安全概率，重写必须按预期方向偏移；（b）通过 NLI 双向蕴含验证语义等价。
   - 最终得到约 17,988 个训练样本，每个样本对应 5 个不同方向的重写，形成响应对。
3. **训练流程（两阶段）**：
   - **阶段一：监督微调（SFT）**。使用成对的重写（低不安全概率 → 高不安全概率）训练模型学习语义保留下的有害性转移。
   - **阶段二：直接偏好优化（DPO）**。从同一原始响应的多个重写中构建偏好三元组（最高、中等、最低不安全概率），优化模型倾向生成更高不确定概率的重写，同时锚定参考策略防止过度偏移。
4. **推理流程**：对于每个受害 LLM 输出，重写器生成候选改写，经语义门验证后返回给攻击者。若验证失败则重新生成。

## 7. 优点

- **范式创新**：首次提出通过**篡改反馈信号**而非检测/阻断来防御多轮越狱，抓住了攻击循环的关键薄弱环节。
- **语义保持**：通过 NLI 双向蕴含确保改写不改变原始响应含义，避免破坏模型正常能力。
- **训练策略有效**：SFT + DPO 两阶段训练，利用相对偏好而非绝对分数，提升跨裁判泛化性。
- **轻量部署**：仅需在 API 边界添加轻量重写器（4B 模型），无需修改受害者 LLM 权重或访问内部表示。
- **实验全面**：涵盖 5 种多轮攻击、多种裁判配置、跨模型转移、自适应攻击消融，结果可信度高。

## 8. 不足与局限

- **成本与延迟**：引入重写器和语义门增加了推理延迟（>5.4 ms/token + 多次重试），在高吞吐场景下可能不可接受。
- **对预优化攻击无效**：若攻击者事先在不使用在线反馈循环的情况下（如其他模型上预优化提示），D-Judge 无法防御。
- **裁判依赖假设**：假设攻击者使用外部裁判模型进行在线优化；若攻击者改用其他反馈机制（如人工反馈、固定规则），D-Judge 可能失效。
- **数据构造偏倚**：训练数据仅来源于 PKU-SafeRLHF 的中等有害响应，可能对新类型或极端有害的响应泛化能力有限。
- **安全税**：尽管 MT-Bench 分数下降不大，但仍有 0.65 分的降低，且在某些子领域（如提取、推理）下降更明显，表明存在一定能力损失。
- **未评估更强自适应攻击**：考虑攻击者知道 D-Judge 存在并尝试逆向工程或绕过重写（如通过对抗性提示诱导重写器失败），文中仅测试了“重写感知评判”这一简单自适应，强度可能不足。
