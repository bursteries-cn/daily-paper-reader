## 摘要
知识编辑系统必须更新选定的事实，同时保持邻近但不相关的行为。本文在记忆辅助设置下研究这一问题，其中推理时检索编辑记忆，并通过参数高效适配器校正模型的物体偏好。我们认为核心设计问题不仅在于如何写入编辑，还在于何时抑制它。我们引入了\method{}，一种路由专用的双适配编辑器。相关性路由器首先决定一个提示是否应该接收编辑记忆。经路由的提示使用一个编辑适配器，该适配器训练为偏好新物体而非原物体；未路由的非直接提示使用一个单独的局部性适配器，训练为保留或恢复原始物体偏好。我们在三个1000例协议\cf{}、\zsre{}和\mquake{}上，采用相同记忆协议和两个7B/8B基础模型评估了\method{}。在Llama-3.1-8B-Instruct上，\method{}在所有三个基准测试中取得了最佳总体概率偏好准确率：\cf{}为0.8180，\zsre{}为0.8946，\mquake{}为0.9922。在Qwen3-8B上趋势相同。路由器消融实验显示，相关记忆边界在不同数据集间存在差异：在\cf{}上词法神经网络路由器最安全，而在\zsre{}和\mquake{}上BGE嵌入路由效果更好。组件和模块消融实验表明，增益主要来自将编辑注入与离路抑制分离，而非简单增加LoRA容量。

## Abstract
Knowledge editing systems must update selected facts while preserving nearby but irrelevant behavior. This paper studies this problem in a memory-assisted setting where an edit memory is retrieved at inference time and a parameter-efficient adapter corrects the model's object preference. We argue that the central design question is not only how to write an edit, but also when to suppress it. We introduce \method{}, a route-specialized dual-adapter editor. A relevance router first decides whether a prompt should receive an edit memory. Routed prompts use an edit adapter trained to prefer the new object over the original object; unrouted non-direct prompts use a separate locality adapter trained to preserve or restore the original-object preference. We evaluate \method{} on three 1,000-case protocols, \cf{}, \zsre{}, and \mquake{}, under the same memory protocol and two 7B/8B base models. On Llama-3.1-8B-Instruct, \method{} obtains the best overall probability-preference accuracy on all three benchmarks: 0.8180 on \cf{}, 0.8946 on \zsre{}, and 0.9922 on \mquake{}. The same trend holds on Qwen3-8B. Router ablations show that the relevant memory boundary differs across datasets: a lexical neural router is safest on \cf{}, while BGE embedding routing is better on \zsre{} and \mquake{}. Component and module ablations show that the gain mainly comes from separating edit injection from off-route suppression rather than from simply increasing LoRA capacity.

## 1. 核心问题与整体含义（研究动机与背景）

大型语言模型（LLM）中存储的事实关联可能过时、错误或与目标任务不一致。知识编辑旨在不重新训练整个模型的情况下更改这些关联。一个理想的编辑器应满足：对编辑后的提示可靠、能泛化到改写句、并保持局部性——即提及相关主体或关系的邻近提示不应继承编辑结果。在**记忆辅助设置**下，推理时检索编辑记忆，并借助参数高效适配器（如LoRA）校正模型的输出偏好。作者指出，核心设计问题不仅在于**如何写入编辑**，还在于**何时抑制编辑**——即编辑记忆被错误地应用于无关提示时，必须抑制其影响。因此，需要同时解决两个关联决策：何时检索编辑记忆，以及检索后激活哪个校正模块。

## 2. 方法论：核心思想、关键技术细节与流程

### 核心思想：路由专用双适配器（RRDA）

- **相关性路由器**：判断输入提示是否与存储的编辑记忆相关。如果相关（直接提示或经路由），则检索记忆并激活**编辑适配器**；如果不相关且为非直接提示，则不检索记忆，激活**局部性适配器**。
- **编辑适配器**：通过交叉熵损失和成对排序损失（新对象 vs 原对象）训练，使模型偏好新对象。
- **局部性适配器**：仅在无记忆的局部性样本上训练，使用排序损失（使原对象保留偏好），以抑制编辑泄露。
- **推理策略**：固定路由策略——直接提示使用编辑适配器（门控值0.75），经路由的非直接提示使用编辑适配器（门控值0.50），未路由的非直接提示使用局部性适配器（门控值0.75）。门控值通过验证集选定。
- **概率偏好度量**：使用长度归一化的对数概率分数，比较新对象与原对象的得分。严格于生成泄露检测。

### 关键技术细节

- 编辑记忆：为每个编辑案例构建一条文本记忆记录。CounterFact用直接提示+新对象；ZsRE用问答形式；MQuAKE-CF用单跳改写拼接。
- 路由器家族：Subject（主体匹配）、Subject+relation（主体加关系词重叠）、NN（哈希词袋MLP，带正负样本训练）、Semantic（BGE嵌入相似度，可选MiniLM重排）。验证集选择最佳路由器（CounterFact选NN，ZsRE和MQuAKE选BGE）。
- 训练目标公式：
  - 编辑适配器损失：\( L_{edit} = \lambda_{ce} L_{ce} + \lambda_{rank} L_{rank} \)，其中 \( L_{rank} = \text{softplus}(m - \text{score}(x, y^+) + \text{score}(x, y^-)) \)。
  - 局部性适配器损失：\( L_{loc} = \text{softplus}(m_{loc} - \text{score}(x, o_0) + \text{score}(x, o_*)) \)。

### 算法流程（文字说明）

1. 构建固定记忆库。
2. 对每个查询提示，路由器判断是否检索记忆。
3. 根据路由结果选择适配器（编辑或局部性）和门控值。
4. 将记忆（如有）与提示拼接，输入冻结的LLM，加上选定的LoRA适配器，计算得分。
5. 最终比较新对象与原对象的得分，判定是否编辑成功。

## 3. 实验设计：数据集、基准、对比方法

### 数据集与场景

- **ZsRE**：1000个案例，关系抽取风格，本地提示为改变关系但保留主体的问题。
- **MQuAKE-CF**：1000个案例，多跳推理的合成编辑，本地提示为跨案例合成。注意：本文使用概率偏好+合成局部性评估，非官方精确匹配生成基准。

- **LocFT-style**：单个LoRA适配器，仅使用直接编辑交叉熵训练（基于强微调基准LocFT-BF）。
- **PRUNE-style**：单个LoRA适配器，使用直接交叉熵+局部性KL正则化（基于扰动抑制编辑PRUNE）。

## 6. 主要结论与发现

1. **RRDA在所有基准和两模型上达到最佳总体概率偏好准确率**。Llama-3.1-8B上：CounterFact 0.8180、ZsRE 0.8946、MQuAKE-CF 0.9922；Qwen3-8B上趋势一致。
2. **增益主要来自分离编辑注入与离路抑制**，而非单纯增加LoRA容量。组件消融显示双适配器政策优于任一单适配器。目标模块消融表明扩大LoRA目标（超出q/v）几乎不改善准确率。
3. **最优路由器依赖数据集**：CounterFact需要避免过度路由邻近提示（NN最好）；ZsRE需要关系敏感路由（BGE最好）；MQuAKE-CF需要语义检索（BGE最好）。
4. **主增益在局部性**：CounterFact和ZsRE上，RRDA的局部性准确率显著高于基线；修复了“同主体不同关系”的失败模式。

## 7. 方法优点

- **设计简洁、可审计**：路由器固定，记忆显式，适配器切换透明，适合需要编辑溯源的场景。
- **参数高效**：仅添加少量LoRA参数（rank-8），训练仅500步，单GPU可运行。
- **概率偏好度量严格**：能捕获渐进式概率泄露，比贪婪解码更敏感。
- **路由专用思想新颖**：将“何时抑制”作为核心问题，与“何时写入”并列，解决了记忆辅助编辑中关键的过泛化问题。

## 8. 不足与局限

- **基线未能涵盖最新丰富方法**：未与ROME、MEMIT、SERAC、MeLLo等主流编辑方法直接对比，仅比较了两种简化PEFT基线。
- **MQuAKE-CF评估非标准**：使用合成局部性和概率偏好，而非官方多跳精确匹配生成，结论推广性受限。
- **CounterFact改写准确率低**：仅0.63左右，表明需要更好的改写增强或路由策略。
- **路由器校准不足**：语义路由器在CounterFact上过度路由，伤害局部性；压力测试显示脆弱性。
- **实验规模有限**：仅7B/8B模型、500步训练；更大模型、更长序列编辑、持续编辑场景未探索。
- **潜在社会风险**：承认编辑可能产生敏感后果，需要授权、日志和审计。
