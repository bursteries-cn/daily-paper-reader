## 摘要
推理多模态大语言模型（MLLMs）的兴起——其在生成答案前会生成显式的思维链（CoT）推理——为知识编辑带来了新挑战：在传统指标（教师强制准确率高达100%）下看似成功的方法，在检查模型推理过程时可能会严重失败（接地成功率低至0%）。我们识别出三种失败模式：（1）结构崩塌，即权重修改方法破坏CoT格式；（2）认知失调，即模型推理链基于视觉证据主动拒绝注入的编辑事实；（3）浅层内化，即方法在精确查询上成功，但在改写或多跳变体上失败。在推理MLLMs中，这些模式相互作用：泛化方法（FT、LoRA）触发格式崩塌，而无深度修改的方法无法泛化。为暴露这些失败，我们提出了一种CoT感知的评估协议，并构建了ReasonEdit-Bench，包含冲突分层、多级探针和多跳可移植性测试。我们提出了CRANE，一种检索增强框架，无需每次编辑进行参数修改。CRANE结合了模态感知的双库检索系统和两阶段训练策略：用于结构初始化的监督微调（SFT），随后是带有认知路由奖励（Cognitive Routing Reward）的GRPO，训练模型在视觉先验和注入的编辑事实之间进行仲裁。在ReasonEdit-Bench上，CRANE在冲突场景中达到96.9%的接地成功率，在多跳链中达到96.9%的中间实体使用率，文本局部性97.6%，图像局部性编辑独立性68.1%。在分布外MMEVOKE基准上，CRANE在黄金检索下达到87.0%。”}

## Abstract
The emergence of reasoning multimodal large language models (MLLMs), which generate explicit chain-of-thought (CoT) reasoning before producing answers, has introduced a new challenge for knowledge editing: methods that appear successful under traditional metrics (teacher-forcing accuracy up to 100%) can fail severely when the model's reasoning process is examined (Grounded Success as low as 0%). We identify three failure modes: (1) Structural Collapse, where weight-modifying methods destroy the CoT format; (2) Cognitive Dissonance, where the model's reasoning chain actively rejects the injected edit fact based on visual evidence; and (3) Shallow Internalization, where methods succeed on exact queries but fail on rephrase or multi-hop variants. On reasoning MLLMs, these modes interact: methods that generalize (FT, LoRA) trigger format collapse, while methods without deep modification cannot generalize. To expose these failures, we propose a CoT-aware evaluation protocol and construct ReasonEdit-Bench, with conflict stratification, multi-level probes, and multi-hop portability tests.   We propose CRANE, a retrieval-augmented framework that requires no per-edit parameter modification. CRANE combines a modality-aware dual-library retrieval system with a two-phase training strategy: Supervised Fine-Tuning (SFT) for structural initialization, followed by GRPO with a Cognitive Routing Reward that trains the model to arbitrate between visual priors and injected edit facts. On ReasonEdit-Bench, CRANE achieves 96.9% Grounded Success on conflict scenarios and 96.9% intermediate entity usage in multi-hop chains, with 97.6% text-locality and 68.1% image-locality Edit Independence. On the out-of-distribution MMEVOKE benchmark, CRANE reaches 87.0% under gold retrieval.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：随着 OpenAI o1、DeepSeek-R1 等推理型大模型的出现，模型在生成最终答案前会输出显式的思维链（CoT），这对知识编辑提出了全新挑战。传统知识编辑方法在推理 MLLM 上存在**虚假成功（Spurious Success）** 现象：在教师强制（Teacher-Forcing）评估下准确率接近 100%，但在自由生成时，模型会通过 <think> 推理过程主动拒绝编辑事实，导致接地成功率（Grounded Success）低至 0%。
- **核心问题**：现有知识编辑方法在推理 MLLM 上存在三种未被传统指标捕获的失效模式：
  1. **结构崩塌（Structural Collapse）**：参数修改方法（FT、LoRA、WISE、GRACE、LiveEdit）严重破坏模型的 <think>…<answer>… 格式，格式化率归零或极低。
  2. **认知失调（Cognitive Dissonance）**：模型的推理链“看到”图像证据后，主动拒绝注入的编辑事实，即使编辑已正确存储。
  3. **浅层内化（Shallow Internalization）**：基于外部记忆的拦截方法（GRACE）在原始查询上成功，但无法泛化到改写、换图或多跳变体。
- **整体含义**：传统教师强制评估不可靠，需要新的评估协议和专门面向推理 MLLM 的知识编辑方法。论文提出 CRANE 框架，通过**无需参数修改的检索增强 + 基于强化学习的认知路由训练**解决上述问题。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- **不修改模型参数**，通过检索系统获取编辑事实，避免结构崩塌。
- **训练模型学会仲裁视觉先验和编辑事实**：通过两阶段训练（SFT + GRPO）使模型能正确引用、覆盖或忽略编辑事实，应对认知失调和浅层内化。

### 2.2 关键技术细节

#### 模态感知双库检索系统
- **双库架构**：独立存储图像索引（SigLIP 嵌入）和文本索引（SigLIP 文本嵌入）。查询时根据模态确定性路由：含图像的查询走图像索引，纯文本走文本索引。
- **对比投影头**：训练轻量线性层将 SigLIP 1152 维嵌入投影至 256 维，使用对称 InfoNCE 损失（公式 (1)），提升同实体不同照片的检索命中率（hit@1 从 69.7% 提升至 90.4%）。
- **置信度阈值防御**：设定阈值 θ=0.55，过滤低相似度匹配（主要针对局部性场景的误检），在保留 98.7% 正确检索的同时降低 33.3% 的假阳性，提升图像局部性。

#### Phase I：监督微调（SFT）初始化
- **数据构造**：使用 Gemini Flash 生成 6,000 条高质量 CoT 示例，覆盖正常、冲突、局部性、文本局部性四类场景，教会模型元技能：识别并引用编辑事实、承认冲突并覆盖、识别无关。
- **训练配置**：冻结视觉 Transformer 和对齐器，仅微调语言模型部分。训练 2 个 epoch，数据平衡（正常 2000、冲突 2000、局部性 1000、文本局部性 1000）。
- **作用**：非注入具体事实，而是建立 CoT 编辑的结构性行为模式。

#### Phase II：基于 GRPO 的认知路由奖励训练
- **GRPO**：生成 G=8 个候选响应，计算组相对优势，使用裁剪 KL 正则化（ϵ=0.2, β=0.05）。
- **总奖励**：R_total = R_accuracy + R_format + R_reasoning（三者等权）。
  - **R_accuracy**：软匹配（F1 或包含关系）。
  - **R_format**：输出是否包含有效 CoT 格式（二值）。
  - **R_reasoning**（核心）：根据样本类型施加不同子奖励：
    - 正常样本 → R_align：奖励在 <think> 中显式引用编辑事实。
    - 冲突样本 → R_override：奖励三阶段模式（承认视觉证据 + 转折词如 "however" + 承诺编辑事实），完整得 1.0，部分得 0.5。
    - 局部性样本 → R_isolate：奖励识别图像/文本与编辑事实无关，禁止幻觉。

## 3. 实验设计：数据集、基准、对比方法

### 3.1 数据集与基准
- **ReasonEdit-Bench**：从 VLKEB 构建，经三条流水线清洗和扩展：
  - 管道 1：视觉接地过滤，去除图像-问题不匹配。
  - 管道 2：冲突严重性分类（OCR 冲突/类别冲突 vs 正常）。
  - 管道 3：自由形式扩展（编辑事实扩展为两句话 + 二级属性探针）。

- **MMEVOKE**：分布外基准（新闻图像 + Wikipedia 实体），评估泛化能力。

### 3.2 对比方法
- **参数修改方法**：Fine-tuning (FT)、LoRA、WISE、GRACE、LiveEdit（单次编辑模式，直接提供编辑事实）。

### 3.3 评估指标（CoT 感知协议）
- **Schema Integrity（格式率）**：规则检查，输出是否包含有效 CoT 格式。
- **Grounded Success (GS)**：LLM 裁判（Gemini-2.5-Flash）检查推理链是否显式基于编辑事实，且答案匹配目标。仅答案正确但未引用编辑事实则判 0。

- **Intermediate Entity Used (IU) / Used Visual Instead (UVI)**：多跳指标，检查编辑实体是否在中间步骤被传播，或模型回归视觉先验。

## 6. 论文的主要结论与发现

1. **传统指标不可靠**：TF-Acc 与 GS 差距在推理 MLLM 上可达 99pp（冲突场景），教师强制完全绕过了推理链。
2. **三种失效模式普遍存在**：FT/LoRA/WISE 导致结构崩塌（格式率 0%）；IKE 等受认知失调影响严重（冲突场景 GS 低至 6.1%）；GRACE 受浅层内化（改写 GS 几乎为 0）。
3. **CRANE 有效解决所有失效模式**：
   - 避免结构崩塌（格式率 99.9%）。
   - 冲突场景 GS 96.9%，甚至高于正常场景（90.5%），因为 R_override 奖励专门训练了覆盖行为。
   - 多跳 IU 96.9%，UVI 仅 1.7%，实现真正知识内化。
   - 保持合理局部性（文本 EI 97.6%，图像 EI 68.1%）。
4. **检索是当前部署瓶颈**：MMEVOKE 上黄金检索 CRANE 达 87.0%，而融合检索仅 69.8%，差距 17pp。
5. **消融表明 SFT 是结构基础，GRPO 的推理奖励主要在局部性上生效**：无 SFT 初始化导致冲突 GS 下降 17.8pp，多跳 IU 下降 61pp；去除推理奖励仅图像 EI 下降 9.7pp，其他不变。

## 7. 优点

- **问题洞察深刻**：首次系统揭示推理 MLLM 上知识编辑的三种隐蔽失效，并设计 CoT 感知指标暴露“虚假成功”。
- **基准贡献**：构建 ReasonEdit-Bench，包含冲突分层、多级内化探针和多跳可移植性，可复用于未来研究。
- **方法创新**：CRANE 通过检索增强避免参数修改（解决结构崩塌），通过双阶段训练和场景条件化奖励（认知路由奖励）动态仲裁视觉与编辑事实，兼具可靠性和局部性。
- **消融完备**：每个组件贡献量明确，验证了设计合理性。
- **跨域泛化**：在分布外 MMEVOKE 上仍优于基模型，验证推理行为的迁移能力。

## 8. 不足与局限

- **图像局部性偏低**（68.1% EI）主要源于检索系统假阳性（12.3% locality false hit），而非推理问题。论文已通过阈值减轻，但仍有提升空间。
- **检索瓶颈显著**：分布外泛化时融合检索仍不鲁棒，黄金检索性能明显优于自动检索，说明检索模块是当前薄弱环节。
- **模型规模有限**：仅在 7B 模型上验证，更大模型（如 70B）的效果和资源需求未知。
- **编辑模式单一**：仅评估单次编辑，未涉及多次累积编辑、编辑顺序影响或事实冲突场景。
- **部分失效模式仅针对推理 MLLM**：结构崩塌和认知失调的结论不适用于标准 MLLM，但浅层内化具有通用性。
- **缺乏计算资源细节**：未报告训练和推理的算力消耗，不利于复现。
- **依赖 GPT/ Gemini 生成数据**：SFT 数据和 LLM Judge 依赖外部模型，可能存在偏差，但论文已做 5% 人工验证（>95% 一致），可信度较高。
